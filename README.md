# Лендинг SIP PRO

## Что собрано

Статический конверсионный лендинг для направления строительства домов из
СИП-панелей. Версия перестроена вокруг одной целевой заявки: клиент выбирает
формат дома, ориентировочную площадь, этажность и оставляет телефон.

Первый экран не показывает стоимость и конкретную квадратуру. Главный акцент:

- дом из СИП-панелей для жизни за городом;
- собственное производство СИП-панелей;
- понятная комплектация;
- быстрый переход к расчету через короткий квиз.

## Файлы

- `index.html` — структура сайта и клиентские тексты.
- `styles.css` — адаптивная верстка, мобильная версия и iOS-style bottom sheet для квиза.
- `script.js` — шапка при скролле, мобильное меню, пошаговый квиз и отправка заявки во внешний шлюз.
- `lead-gateway/` — отдельная Yandex Cloud Function для доставки заявок в рабочую группу MAX.
- `assets/` — изображения для первого экрана, карточек проектов и этапов строительства.

## Структура сайта

- первый экран: оффер без цен, с CTA `Заказать расчет`;
- короткий блок выгод СИП-дома;
- проекты и варианты с кнопками `Хочу такой`;
- комплектация `Что вы приобретаете у нас`;
- этапы строительных работ;
- блок доверия про собственное производство СИП-панелей;
- FAQ;
- контакты и быстрый переход к заявке.

## Логика заявки

Квиз сделан как всплывающая мобильная форма:

- шаг 1: для кого строится дом;
- шаг 2: желаемая площадь и этажность;
- шаг 3: телефон.

После успешной доставки в шлюз показывается экран успешной заявки:
`Поздравляем, заявка принята`.

В форме нет имени, выбора мессенджера, времени звонка, расчетов стоимости и
публичных ориентиров по цене. Все детали уточняются на звонке.

Публичный сайт не хранит токен MAX-бота. В `index.html` задается только
`window.SIP_LEAD_ENDPOINT`; токен бота и ID группы указываются в переменных
окружения Cloud Function.

## Что осталось подключить

- Реальные ссылки на WhatsApp, Max, Telegram и звонок в контактном блоке.
- Создать отдельного MAX-бота для SIP PRO и добавить его в рабочую группу.
- Опубликовать `lead-gateway/` в Yandex Cloud Function и указать реальный endpoint в `index.html`.
- Реальные фото объектов и производства, если они есть у заказчика.

## Проверки перед публикацией

- мобильная версия от 320px;
- отсутствие горизонтальной прокрутки;
- видимость карточек и изображений;
- открытие квиза с кнопок CTA;
- прохождение квиза до экрана успешной заявки;
- отсутствие цен и конкретной квадратуры на первом экране.
