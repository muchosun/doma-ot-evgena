# Шлюз заявок квиза: Yandex Cloud -> MAX

Этот модуль принимает заявку с квиза сайта `sipdom-krd.ru`, проверяет поля и отправляет сообщение в рабочую группу MAX. Он отделён от статичного сайта: токен бота и ID группы не попадают в GitHub Pages, исходный код страницы или Яндекс.Метрику.

## Что принимает и отправляет

`POST /v1/leads` принимает только запросы с разрешённых доменов из `LEAD_ORIGINS` и только JSON.

В MAX придёт:

- телефон клиента;
- сценарий строительства: для себя, для семьи или индивидуальное решение;
- желаемая площадь;
- этажность;
- путь страницы, с которой оставили заявку;
- нативная карточка контакта MAX с номером клиента.

`GET /health` возвращает технический статус без персональных данных.

## Безопасность

- `MAX_BOT_TOKEN` и `MAX_CHAT_ID` задаются только как переменные окружения Cloud Function. Их нельзя добавлять в `index.html`, `script.js`, GitHub Secrets для Pages или Telegram.
- Для обращения к `platform-api2.max.ru` функция использует официальный корневой и выпускающий TLS-сертификаты Минцифры из `certs/russiantrustedca.pem`. Проверка TLS не отключается.
- Для упоминаний ответственных можно заполнить `MAX_MANAGER_USER_ID`, `MAX_MANAGER_NAME`, `MAX_EVGENIY_USER_ID` и `MAX_EVGENIY_NAME`. Если переменные не заполнены, заявка всё равно будет доставлена, но без упоминаний.
- Функция не записывает номер телефона в логи. В логах остаётся только технический ID доставки.
- Перед запуском платного трафика нужно добавить SmartCaptcha в API Gateway либо в форму. Проверка `Origin` и скрытое поле-ловушка не заменяют защиту от ботов.

## Публикация в Yandex Cloud

1. Создать отдельного чат-бота MAX для SIP PRO и добавить его в рабочую группу MAX.
2. Получить `MAX_CHAT_ID` группы через `scripts/find-max-chat-id.js`.
3. Создать Cloud Function на `nodejs22`, загрузить содержимое этой папки и указать точку входа `index.handler`.
4. В переменных окружения функции указать значения из `.env.example`:
   - `LEAD_ORIGINS=https://sipdom-krd.ru,https://www.sipdom-krd.ru`;
   - `MAX_API_BASE=https://platform-api2.max.ru`;
   - `MAX_BOT_TOKEN` — токен отдельного чат-бота MAX;
   - `MAX_CHAT_ID` — ID рабочей группы MAX, куда бот уже добавлен.
5. Создать API Gateway по `openapi.template.yaml`, заменив `<FUNCTION_ID>` и `<SERVICE_ACCOUNT_ID>` на реальные идентификаторы.
6. Открыть `https://<gateway-domain>/health`: ожидаемый ответ `{"status":"ok"}`.
7. После проверки тестовой заявки добавить в корневой `index.html` сайта публичную настройку:

   ```html
   <script>
     window.SIP_LEAD_ENDPOINT = "https://<gateway-domain>/v1/leads";
   </script>
   ```

   Этот блок должен стоять до подключения `script.js`.

## Локальная проверка

В этой папке:

```bash
npm test
```

Тесты не отправляют реальные сообщения, не требуют токена и проверяют CORS, валидацию, формат сообщения и формат запроса в MAX.

## Однократное получение ID рабочего чата MAX

MAX не позволяет получить список чатов бота через `GET /chats`. Для разовой технической настройки есть скрипт `scripts/find-max-chat-id.js`: он выполняет только `GET /updates` и ищет события `bot_added` и `message_created`. Сообщения в MAX он не отправляет.

1. Убедитесь, что у бота нет активной webhook-подписки.
2. Добавьте бота в рабочую группу MAX или отправьте новое сообщение в группе, где бот уже состоит.
3. В терминале введите токен скрыто, затем запустите скрипт:

   ```bash
   read -rs 'MAX_BOT_TOKEN?Токен MAX: '; echo; export MAX_BOT_TOKEN
   node scripts/find-max-chat-id.js
   unset MAX_BOT_TOKEN
   ```

4. Скрипт выведет числовой ID чата. Его нужно добавить в переменную окружения функции `MAX_CHAT_ID`.

## Проверка после публикации

```bash
curl -i https://<gateway-domain>/health

curl -i -X POST https://<gateway-domain>/v1/leads \
  -H 'Origin: https://sipdom-krd.ru' \
  -H 'Content-Type: application/json' \
  --data '{"purpose":"Дом для семьи","area":72,"floors":"1 этаж","phone":"+7 999 123-45-67","source":"https://sipdom-krd.ru/#quiz"}'
```

Ожидаемый результат: `201`, затем одно сообщение в MAX. При `400` исправляется payload, при `403` проверяются домены в `LEAD_ORIGINS`, при `502` — токен бота, ID чата и доступ бота к рабочей группе.
